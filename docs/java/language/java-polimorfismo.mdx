# Java Polimorfismo

Este ejemplo muestra **polimorfismo en Java** usando una `interface` (`Shape`) y varias implementaciones concretas (`Circle` y `Rectangle`).  
Es útil para entender cómo programar contra **abstracciones** (principio de programación a interfaces, relacionado con SOLID).

## Código de ejemplo

Los archivos del ejemplo se encuentran en `examples/java-polimorfismo`:

```java title="Shape.java"
public interface Shape {
  double area();
}
```

```java title="Circle.java"
public class Circle implements Shape {
  private final double radius;

  public Circle(double radius) {
    this.radius = radius;
  }

  @Override
  public double area() {
    return Math.PI * radius * radius;
  }
}
```

```java title="Rectangle.java"
public class Rectangle implements Shape {
  private final double width;
  private final double height;

  public Rectangle(double width, double height) {
    this.width = width;
    this.height = height;
  }

  @Override
  public double area() {
    return width * height;
  }
}
```

```java title="PolymorphismExample.java"
public class PolymorphismExample {
  public static void main(String[] args) {
    Shape circle = new Circle(2.0);
    Shape rectangle = new Rectangle(3.0, 4.0);

    printArea(circle);
    printArea(rectangle);
  }

  private static void printArea(Shape shape) {
    System.out.println("Área: " + shape.area());
  }
}
```

Puntos clave:

- `Shape` define la **abstracción** que todas las figuras deben cumplir.
- `Circle` y `Rectangle` implementan la misma interface pero calculan el área de forma distinta.
- `printArea` recibe un `Shape`, no una implementación concreta: ahí es donde se ve el **polimorfismo**.

## Cómo ejecutarlo localmente

Requisitos: **JDK 17+ (o 11+)** instalado y disponible en tu `PATH`.

En una terminal, dentro del proyecto ya clonado:

```bash
cd examples/java-polimorfismo
javac *.java
java PolymorphismExample
```

Deberías ver una salida similar a:

```text
Área: 12.566370614359172
Área: 12.0
```
